# ЯсноТекст

> Программа, которая превращает «недоступные» цифровые документы
> в удобную среду для людей с нарушениями зрения и дислексией.

Учебный проект по дисциплине «Основы проектной деятельности».

## Проблема

Миллионы цифровых документов — учебники в PDF, сканы, методички в DOCX —
недоступны для людей с особенностями зрения или дислексией. Часто это
просто картинки, которые невозможно ни увеличить без потери качества,
ни перекрасить, ни озвучить стандартными программами.

Для человека с особенностями восприятия такой файл — это запертая дверь
к знаниям. ЯсноТекст помогает её открыть.

## Решение

ЯсноТекст работает как «умный посредник» между документом и читателем:

1. Открывает PDF, DOCX и изображения (PNG/JPG/TIFF/BMP).
2. Если у PDF нет текстового слоя — автоматически распознаёт страницы
   через OCR (Tesseract).
3. Применяет один из трёх встроенных профилей доступности или ваш
   собственный — и весь интерфейс адаптируется под потребности.
4. По желанию озвучивает текст голосовым синтезатором с подсветкой
   текущего предложения.

## Возможности

### Чтение документов

- **PDF** с текстовым слоем — через [PdfPig](https://github.com/UglyToad/PdfPig)
  с правильным извлечением слов (ContentOrderTextExtractor).
- **PDF-сканы** — рендерятся в bitmap'ы через PDFium, страницы прогоняются
  через OCR.
- **DOCX** — через `DocumentFormat.OpenXml`, поддерживает страницы
  и параграфы.
- **Изображения** (PNG/JPG/JPEG/TIFF/BMP) — OCR Tesseract на лету.

### Адаптация

- **Три встроенных профиля**:

  | Профиль | Шрифт | Размер | Цветовая схема | Особенности |
  |---|---|---|---|---|
  | Стандартный | Segoe UI | 14pt | Чёрный на белом | Базовый режим |
  | Слабовидение | Arial Bold | 28pt | Жёлтый на чёрном | Контраст 21:1, крупные элементы |
  | Дислексия | OpenDyslexic | 18pt | Тёмно-коричневый на бежевом | Расширенный межстрочный, специальный шрифт |

- **Пользовательские профили** — до 7 штук, сохраняются в
  `%APPDATA%/YasnoText/profiles.json`. Можно создавать на основе
  любого активного, переименовывать, перезаписывать, удалять,
  переупорядочивать drag-and-drop (встроенные при этом строго
  закреплены сверху).
- **Шрифт и межстрочный интервал** меняются на лету слайдером и
  кнопками A+/A−. Список шрифтов ограничен теми, что точно
  поддерживают кириллицу.
- **Темы** переключаются мгновенно для всего интерфейса
  через DynamicResource.

### Озвучка

- Старт/Пауза/Стоп для всего открытого текста через
  `System.Speech.Synthesis`.
- Автоматически выбирается русский голос, если установлен
  (Pavel/Irina на Windows 11).
- Подсветка текущего предложения по событию SpeakProgress —
  Run-уровневая, не блокирует скролл.

### UX-мелочи

- **Drag-and-drop** файлов в окно — открывается через тот же
  пайплайн, что и диалог.
- **Список недавних файлов** (до 10) в меню «Файл → Недавние»,
  LRU, удалённые с диска убираются из списка.
- **Режим чтения** (F11) — скрывает меню/toolbar/sidebar/status bar,
  оставляя только текст. Подсказка о выходе тает через 3 секунды,
  возвращается на mouse-move. Esc или клик по плашке тоже выходит.
- **Автопрокрутка** по щелчку средней кнопкой мыши, как в браузерах:
  курсор задаёт скорость, Esc/любой клик выключает.
- **Заголовок окна** показывает имя открытого файла.
- **Окно «Горячие клавиши»** в меню «Справка».

## Горячие клавиши

| Действие | Сочетание |
|---|---|
| Открыть документ | `Ctrl+O` |
| Закрыть документ | `Ctrl+W` |
| Сохранить как новый профиль | `Ctrl+S` |
| Профиль «Стандартный» / «Слабовидение» / «Дислексия» | `Ctrl+1` / `Ctrl+2` / `Ctrl+3` |
| Увеличить / уменьшить шрифт | `Ctrl+=` / `Ctrl+−` |
| Озвучить (старт / пауза / продолжить) | `F5` |
| Остановить озвучку | `Shift+F5` |
| Войти / выйти из режима чтения | `F11` |
| Выйти из автопрокрутки | `Esc` |

## Технологии

- **.NET 10** + **WPF** — платформа и UI
- **PdfPig** + **PdfPig.DocumentLayoutAnalysis** — парсинг PDF
- **DocumentFormat.OpenXml** — парсинг DOCX
- **Tesseract** — OCR (`eng`+`rus`)
- **PDFium** (через PDFtoImage) — рендеринг PDF-страниц для OCR
- **System.Speech.Synthesis** — озвучка
- **OpenDyslexic** — шрифт для профиля «Дислексия», встроен как
  Resource в .exe
- **xUnit** — юнит-тесты (91 тест на момент релиза)

## Сборка и запуск

**Требования:**

- Windows 10 или 11
- [.NET 10 SDK](https://dotnet.microsoft.com/download)

**Запуск из исходников:**

```bash
git clone https://github.com/Serouke/yasnotext.git
cd yasnotext
dotnet run --project src/YasnoText.UI
```

### Установка языковых моделей Tesseract

Без этого OCR-функции (открытие PDF-сканов и изображений) не работают.
Файлы крупные (~4 МБ каждый), поэтому в репозитории их нет.

1. Создайте папку `tessdata/` в корне репозитория.
2. Скачайте туда два файла:
   - [eng.traineddata](https://github.com/tesseract-ocr/tessdata_fast/raw/main/eng.traineddata)
   - [rus.traineddata](https://github.com/tesseract-ocr/tessdata_fast/raw/main/rus.traineddata)
3. Соберите проект — MSBuild сам скопирует их в `bin/.../tessdata/`
   рядом с `.exe`.

При запуске без tessdata приложение работает, но при попытке открыть
скан или изображение сообщит понятным MessageBox: «Папка с языковыми
моделями Tesseract не найдена».

## Известные ограничения

- **Математические PDF** (формулы, индексы, степени) — извлекаются
  как набор отдельных символов, потому что PDF хранит математику как
  расположение глифов, а не как структуру. Это фундаментальное
  ограничение формата, не баг парсера.
- **Таблицы и графики** не восстанавливаются как структура — текст
  внутри ячеек извлекается «как есть».
- **Постраничная навигация** (переход к странице, виртуализация по
  страницам) пока не реализована — `FlowDocumentScrollViewer`
  справляется со скроллом на больших документах за счёт встроенной
  виртуализации параграфов.
- **Letter spacing** не применяется к области чтения — WPF
  TextBlock/FlowDocument не поддерживает его нативно. Поле в модели
  профиля оставлено на будущее.
